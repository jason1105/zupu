# 集成测试文档

**版本**：1.0.0  
**日期**：2026-04-18  
**测试框架**：Vitest 4.x + Prisma Client（测试数据库）  
**运行命令**：`DATABASE_URL="file:/tmp/zupu-test.db" npx vitest run tests/integration/`

---

## 1. 测试策略

集成测试分两个层次：

**层次一：数据库集成测试**（`Prisma-backed DB tests`）  
直接使用 Prisma Client 操作独立测试数据库（`/tmp/zupu-test.db`），验证数据库 schema、约束和业务数据流的完整性。

**层次二：树构建集成测试**（`Tree building integration`）  
用完整的家族场景数据（包含所有关系类型）验证 `buildTree` + `flattenTree` 的综合行为。

## 3. 树构建集成测试用例

场景：三代六口之家  
```
祖父(gp) ─── 祖母(gm)
    │
  父亲(fa) ─── 母亲(mo)
    ├── 长子(s1)
    └── 长女(d1)
```

| 编号 | 测试用例 | 验证内容 |
|------|---------|---------|
| TI-01 | 根节点识别 | 祖父是根节点；父亲和儿子不是根节点 |
| TI-02 | 配偶关系分配 | 祖母在祖父的 spouses 中；母亲在父亲的 spouses 中 |
| TI-03 | 扁平化树包含所有成员 | flattenTree 包含祖父、父亲、长子、长女 |
| TI-04 | 父亲有 2 个子女节点 | father.children.length === 2 |

---

## 4. 测试数据管理

- **前置清理（beforeAll）**：删除所有测试数据（前缀 `test-integration-` 的用户、以及关联数据），确保测试幂等
- **后置清理（afterAll）**：同样清理，防止测试污染其他环境
- **测试隔离**：所有 DB 测试在独立的 `/tmp/zupu-test.db` 数据库中进行，不影响开发数据库
