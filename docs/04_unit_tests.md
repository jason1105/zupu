## 2. tree-utils 单元测试

### 2.1 `buildTree`

| 测试用例 | 描述 | 预期结果 |
|---------|------|----------|
| 空输入 | members=[], relationships=[] | 返回空数组 |
| 单个孤立成员 | 无任何关系 | 返回长度为 1 的根节点，children=[], spouses=[] |
| 父子关系 | 1个父节点+1个子节点 | 父为根节点，children 中含子节点 |
| 夫妻+亲子 | 父+母+子，父子PARENT_CHILD，父母SPOUSE | 母出现在父节点的spouses列表，不出现在children中 |
| 三代家庭 | 祖父→父亲→儿子 | 祖父为根，层层嵌套 |
| 多根节点 | 两个无关联成员 | 返回长度为 2 的数组 |
| 循环关系防护 | A是B的父亲，B是A的父亲 | 不抛异常，不死循环 |
| 多子女 | 一父三子女 | children.length === 3 |

### 2.2 `flattenTree`

| 测试用例 | 描述 | 预期结果 |
|---------|------|----------|
| BFS 顺序 | 三代家庭 | 返回 [祖父, 父亲, 儿子] |
| 空输入 | roots=[] | 返回空数组 |

### 2.3 `getMemberDepth`

| 测试用例 | 描述 | 预期结果 |
|---------|------|----------|
| 根节点深度 | 单个根节点 | 0 |
| 嵌套深度 | 三代家庭中各成员 | 祖父=0, 父亲=1, 儿子=2 |
| 不存在的成员 | id 不在树中 | -1 |

## 3. API 辅助逻辑单元测试

### 3.1 成员输入验证

| 测试用例 | 描述 | 预期结果 |
|---------|------|----------|
| 有效性别值 | MALE/FEMALE/OTHER | 均通过验证 |
| 无效性别值 | male/M/''/UNKNOWN | 均被拒绝 |
| 姓名非空 | '张三' | 通过 |
| 姓名为空/空格 | ''/'' '' | 被拒绝 |

### 3.2 关系输入验证

| 测试用例 | 描述 | 预期结果 |
|---------|------|----------|
| 有效关系类型 | PARENT_CHILD/SPOUSE | 通过 |
| 自我关联 | fromId === toId | 被拒绝 |
| 不同成员 | fromId !== toId | 通过 |

### 3.3 密码验证

| 测试用例 | 描述 | 预期结果 |
